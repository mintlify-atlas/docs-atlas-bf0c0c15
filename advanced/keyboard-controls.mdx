## Overview

ngx-stories includes comprehensive **keyboard navigation** support, making your stories accessible to desktop users and improving overall usability.

<Info>
  Keyboard controls are **always enabled** and work alongside touch/mouse gestures.
</Info>

## Keyboard Shortcuts

All keyboard controls are handled in the `handleKeyPress` method (from `ngx-stories.component.ts:164-176`):

```typescript
@HostListener('document:keydown', ['$event'])
handleKeyPress(event: KeyboardEvent): void {
  if (event.key === 'ArrowRight') {
    this.navigateStory('next');
  } else if (event.key === 'ArrowLeft') {
    this.navigateStory('previous');
  } else if (event.key === ' ') {
    event.preventDefault(); // Prevent page scrolling
    this.togglePause();
  } else if (event.key === 'Escape') {
    this.onExit();
  }
}
```

## Detailed Behavior

### Arrow Right → Next Story

**What happens**:
1. Current video pauses (if applicable)
2. Progress bar resets
3. Story index increments
4. If last story in group: moves to first story of next group
5. If last story in last group: triggers `triggerOnEnd` event

**Code flow** (from `ngx-stories.component.ts:393-428`):

```typescript
navigateStory(direction: 'next' | 'previous') {
  if (this.isTransitioning) return;
  this.pauseCurrentVideo(true);
  this.setTransitionState(true);
  this.clearProgressInterval();
  this.clearErrorTimeout();

  const { storyGroupIndex, storyIndex } = this.stateService.navigate(
    direction, 
    this.storyGroupChange.bind(this)
  );

  this.currentStoryGroupIndex = storyGroupIndex;
  this.currentStoryIndex = storyIndex;
  this.resetBackground();

  if (this.currentStoryGroupIndex === this.storyGroups.length) {
    this.onEnd();
    return;
  }

  this.progressWidth = 0;
  this.setTransitionState(false, this.HOLD_DELAY_MS);
  this.startStoryProgress();

  if (this.storyState !== 'paused') {
    this.storyState = 'playing';
    this.cdr.detectChanges();
    this.playCurrentStoryVideo();
  } else {
    this.cdr.detectChanges();
  }
}
```

**Transition state**:
- Prevents rapid successive navigation
- Duration: 500ms (`HOLD_DELAY_MS`)
- Blocks all navigation during transition

### Arrow Left ← Previous Story

**What happens**:
1. Same as next story, but in reverse
2. If first story in group: moves to last story of previous group
3. If first story in first group: stays on first story

**Navigation logic** (from `story-utility.service.ts:75-111`):

```typescript
prevStory(
  storyGroups: StoryGroup[],
  currentStoryGroupIndex: number,
  currentStoryIndex: number,
  onStoryGroupChange: (storyGroupIndex: number) => void
): { storyGroupIndex: number; storyIndex: number } {
  let stories = storyGroups[currentStoryGroupIndex]?.stories;

  if (!stories || stories.length === 0) {
    if (currentStoryGroupIndex > 0) {
      currentStoryGroupIndex--;
      currentStoryIndex = 0;
      onStoryGroupChange(currentStoryGroupIndex);
    }
    return { storyGroupIndex: currentStoryGroupIndex, storyIndex: currentStoryIndex };
  }

  if (currentStoryIndex === 0) {
    if (currentStoryGroupIndex > 0) {
      currentStoryGroupIndex--;
      stories = storyGroups[currentStoryGroupIndex]?.stories;
      if (!stories || stories.length === 0) {
        currentStoryIndex = 0;
      } else {
        currentStoryIndex = stories.length - 1;
      }
      onStoryGroupChange(currentStoryGroupIndex);
    }
  } else {
    currentStoryIndex--;
  }
  return { storyGroupIndex: currentStoryGroupIndex, storyIndex: currentStoryIndex };
}
```

### Space - Toggle Pause/Play

**When paused**:
- Video stops playing
- Progress bar freezes
- Story timer pauses
- Gradient updates stop (for videos)

**When resumed**:
- Video resumes from current position
- Progress continues
- Story timer resumes
- Gradient updates restart (for videos)

**Implementation** (from `ngx-stories.component.ts:582-592`):

```typescript
togglePause() {
  this.storyState = this.storyState === 'paused' ? 'playing' : 'paused';
  if (this.storyState === 'paused') {
    this.clearProgressInterval();
    this.pauseCurrentVideo();
  } else {
    this.startStoryProgress();
    this.playCurrentStoryVideo();
  }
  this.cdr.detectChanges();
}
```

<Note>
  The `event.preventDefault()` call prevents the browser from scrolling the page when Space is pressed.
</Note>

### Escape - Exit Stories

**What happens**:
1. `triggerOnExit` event emits
2. Your application handles the exit (typically closes the viewer)
3. Progress and video playback stop automatically

**Implementation** (from `ngx-stories.component.ts:623-626`):

```typescript
private onExit() {
  this.triggerOnExit.emit();
}
```

**Usage example**:

```typescript
export class AppComponent {
  showStories = true;

  onStoriesExit() {
    this.showStories = false;
  }
}
```

```html
<ngx-stories 
  *ngIf="showStories"
  [storyGroups]="storyGroups"
  (triggerOnExit)="onStoriesExit()">
</ngx-stories>
```

## Edge Cases

### Simultaneous Input

**Question**: What if user presses multiple keys rapidly?

**Answer**: The `isTransitioning` flag prevents overlapping navigation:

```typescript
navigateStory(direction: 'next' | 'previous') {
  if (this.isTransitioning) return; // Blocks overlapping calls
  // ...
}
```

### Paused State Navigation

Navigating to a new story while paused:
- New story loads but remains paused
- User must press Space to resume
- Prevents unwanted auto-play

From `ngx-stories.component.ts:419-427`:

```typescript
if (this.storyState !== 'paused') {
  this.storyState = 'playing';
  this.cdr.detectChanges();
  this.playCurrentStoryVideo();
} else {
  this.cdr.detectChanges(); // Load content but stay paused
}
```

### End of Stories

Pressing Arrow Right on the last story:
- Triggers `triggerOnEnd` event
- Does NOT loop back to first story
- Navigation is blocked

```typescript
if (this.currentStoryGroupIndex === this.storyGroups.length) {
  this.onEnd();
  return; // Stops further execution
}
```

### Focus Requirements

<Warning>
  Keyboard controls use `document:keydown`, meaning they work **globally** without needing focus on the stories component.
</Warning>

This is intentional for better UX - users don't need to click the stories first.

## Disabling Keyboard Controls

Currently, keyboard controls **cannot be disabled** via configuration. If you need to disable them, you have two options:

### Option 1: Fork and Modify

Remove or comment out the `@HostListener`:

```typescript
// @HostListener('document:keydown', ['$event'])
// handleKeyPress(event: KeyboardEvent): void {
//   // ...
// }
```

### Option 2: Intercept Key Events

Prevent keyboard events from reaching the component:

```typescript
export class AppComponent {
  @HostListener('document:keydown', ['$event'])
  handleKeyPress(event: KeyboardEvent) {
    // Block specific keys from reaching ngx-stories
    if (['ArrowLeft', 'ArrowRight', ' ', 'Escape'].includes(event.key)) {
      event.stopPropagation();
      event.preventDefault();
    }
  }
}
```

<Note>
  A future version may include an `options.enableKeyboardControls` flag. Consider opening a feature request on [GitHub](https://github.com/Gauravdarkslayer/ngx-stories).
</Note>

## Customizing Keyboard Shortcuts

To customize shortcuts, fork the library or wrap it:

### Custom Wrapper Example

```typescript
export class CustomStoriesComponent {
  @ViewChild(NgxStoriesComponent) storiesComponent!: NgxStoriesComponent;

  @HostListener('document:keydown', ['$event'])
  handleCustomKeys(event: KeyboardEvent) {
    switch(event.key) {
      case 'n': // Use 'n' for next
        this.storiesComponent.navigateStory('next');
        event.preventDefault();
        break;
      case 'p': // Use 'p' for previous
        this.storiesComponent.navigateStory('previous');
        event.preventDefault();
        break;
      case 'Enter': // Use Enter to pause
        this.storiesComponent.togglePause();
        event.preventDefault();
        break;
    }
  }
}
```

<Warning>
  You'll need to make methods like `navigateStory` and `togglePause` public in a forked version, as they're currently private.
</Warning>

## Accessibility Considerations

### Screen Readers

Keyboard navigation is essential for screen reader users:

- **Arrow keys**: Provide non-touch navigation
- **Space**: Allows users to pause and digest content
- **Escape**: Clear exit path from the viewer

### Best Practices

## Testing Keyboard Controls

### Manual Testing Checklist

- [ ] Arrow Right advances through all stories
- [ ] Arrow Left goes back through all stories  
- [ ] Space pauses and resumes correctly
- [ ] Escape triggers exit event
- [ ] No double-navigation when pressing keys rapidly
- [ ] Works while video is playing
- [ ] Works while story is paused
- [ ] Gradient updates stop when paused (for videos)
- [ ] Progress bar freezes when paused
- [ ] Video audio respects pause state

### Automated Testing

```typescript
import { ComponentFixture } from '@angular/core/testing';

describe('NgxStoriesComponent Keyboard Controls', () => {
  let component: NgxStoriesComponent;
  let fixture: ComponentFixture<NgxStoriesComponent>;

  it('should navigate to next story on ArrowRight', () => {
    const initialIndex = component.currentStoryIndex;
    const event = new KeyboardEvent('keydown', { key: 'ArrowRight' });
    document.dispatchEvent(event);
    fixture.detectChanges();
    expect(component.currentStoryIndex).toBe(initialIndex + 1);
  });

  it('should toggle pause on Space', () => {
    const initialState = component.storyState;
    const event = new KeyboardEvent('keydown', { key: ' ' });
    document.dispatchEvent(event);
    fixture.detectChanges();
    expect(component.storyState).not.toBe(initialState);
  });

  it('should emit exit event on Escape', (done) => {
    component.triggerOnExit.subscribe(() => {
      expect(true).toBe(true);
      done();
    });
    const event = new KeyboardEvent('keydown', { key: 'Escape' });
    document.dispatchEvent(event);
  });
});
```

## Performance

Keyboard event handling has **negligible performance impact**:

- Event listener: ~0.1ms per keypress
- Navigation logic: ~5-10ms
- No continuous polling or intervals
- Events only fire when keys are actually pressed

## Common Issues

### Keys Not Working

**Possible causes**:
1. Another component is intercepting key events
2. Browser extension blocking events
3. Focus trapped in an input field

**Debug**:
```typescript
@HostListener('document:keydown', ['$event'])
handleKeyPress(event: KeyboardEvent): void {
  console.log('Key pressed:', event.key); // Add this
  // ... rest of the code
}
```

### Space Scrolls Page

**Should not happen** - the component calls `event.preventDefault()`

If it still scrolls:
```typescript
if (event.key === ' ') {
  event.preventDefault();
  event.stopPropagation(); // Add this
  this.togglePause();
}
```

### Arrow Keys Move Focus

**Expected in some contexts**. If problematic:
```typescript
if (event.key === 'ArrowRight') {
  event.preventDefault(); // Add this
  this.navigateStory('next');
}
```
